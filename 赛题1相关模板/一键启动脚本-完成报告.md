## 技术实现细节

### 脚本架构

```bash
start_system.sh
├── 配置变量（目录、端口、模型路径等）
├── 工具函数
│   ├── check_process()    # 检查进程是否运行
│   ├── stop_process()     # 停止进程
│   ├── start_webui()      # 启动WebUI服务器
│   ├── start_detection()  # 启动检测程序
│   ├── stop_all()         # 停止所有服务
│   ├── show_status()      # 显示状态
│   └── show_logs()        # 显示日志
└── 主程序（命令路由）
```

### WebUI技术栈

**后端**:
- Python 3 `http.server.SimpleHTTPRequestHandler`
- 端口: 8002
- 绑定: 0.0.0.0（允许外部访问）

**前端**:
- HTML5 + CSS3 + JavaScript
- 无需额外框架
- 数据源: `state.json` (JSON格式状态数据)
- 视频源: `frame.jpg` (实时更新的JPEG图像)
- 刷新频率: 500ms

**数据流**:
```
unified_monitor.py → webui_http_unified/
                     ├── frame.jpg (视频帧)
                     └── state.json (状态数据)
                             ↓
                     浏览器通过 fetch() 自动拉取
```

### 检测程序参数

启动命令：
```bash
python3 unified_monitor.py \
    --yolo-model "runs/detect/train_electro61/weights/yolov8_electro61_aipp.om" \
    --data-yaml "config/electro61.yaml" \
    --danger-distance 300 \
    --conf-thres 0.55
```

### 进程管理

**PID文件**:
- WebUI: `~/ICT/pids/webui.pid`
- 检测程序: `~/ICT/pids/detection.pid`

**日志文件**:
- WebUI: `~/ICT/logs/webui.log`
- 检测程序: `~/ICT/logs/detection.log`

**后台运行**: 使用 `nohup` + `&` 实现守护进程

**优雅关闭**:
1. 首先发送 `SIGTERM` 信号
2. 等待2秒
3. 如果进程仍在运行，发送 `SIGKILL` 强制终止
4. 清理PID文件
